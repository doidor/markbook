# style-markbook

Apply a named visual preset to a Markbook site. Each preset is a self-contained
CSS file that overrides Markbook's `--mb-*` design tokens for both light and
dark modes; no `BASE_CSS` opt-out, no `transformHtml` rewriting — Layer 1 of
the [three-layer customization model](../../../DECISIONS.md) (ADR-0017).

## Available presets

Listed in `presets/`. Each `.css` file is the entire stylesheet; opening one
shows exactly what every preset does.

| Preset | Feel | Accent | Notes |
| --- | --- | --- | --- |
| [`minimal`](presets/minimal.css) | Quiet, low-contrast, narrow | `#444` | Achromatic. Best for prose-heavy docs. |
| [`vibrant`](presets/vibrant.css) | Bold, generous spacing, modern | `#7c3aed` | Strong purple. Bigger headings. |
| [`corporate`](presets/corporate.css) | Traditional, dense | `#1e40af` | Muted blue/gray. Conservative type. |
| [`github`](presets/github.css) | Mimics GitHub Docs | `#0969da` | System font stack, blue accent. |
| [`nord`](presets/nord.css) | Cool, calm, designer-y | `#5e81ac` | Full Nord palette. |

## Inputs

- **`preset`** (required, positional) — one of the names above.
- **`--accent <hex>`** (optional) — override the accent color on top of the preset. E.g. `vibrant --accent '#ff6b6b'`.
- **`--font <family>`** (optional) — override the sans-serif font family. Wrap multi-word names in quotes: `--font '"JetBrains Sans", system-ui'`.
- **`--dest <path>`** (optional) — output path relative to the project root. Defaults to `./markbook.css`.

## Steps

1. **Locate the project root.** `view markbook.config.ts` from the cwd; fail with a clear message if not found.
2. **Validate the preset name** against the table above.
3. **Read the preset file** from `<this skill dir>/presets/<preset>.css`.
4. **Apply overrides if requested.**
   - If `--accent` was supplied, replace the `--mb-accent:` value in both `:root` and `:root[data-theme="dark"]` blocks with the supplied hex. Also update `--mb-link` to match (the two are usually paired).
   - If `--font` was supplied, replace the `--mb-font-sans:` value the same way.
5. **Write the (possibly mutated) preset CSS** to `<dest>`. Default: `./markbook.css`. If the destination file already exists and was generated by a previous run (look for the `/* markbook style preset: ... */` header), overwrite. Otherwise prompt for confirmation before overwriting.
6. **Wire it into `markbook.config.ts`.** If the file already has a `css:` field:
   - If `<dest>` is already in the array, do nothing.
   - Otherwise append the path to the array.

   If the config has no `css:` field, insert one inside the `defineConfig({ ... })` block: `css: ['./markbook.css'],`.
7. **Run [`/verify-build`](../verify-build/SKILL.md)** to confirm the rebuild succeeds. Open the rendered `dist/index.html` is out of scope (no browser); the user can spot-check visually.
8. **Run [`/progress-log`](../progress-log/SKILL.md)** ONLY if this is the first time a preset is being applied or the preset changed. Tweaking the accent on an already-applied preset doesn't warrant a journal entry.

## Adding a new preset

1. Copy an existing `presets/<name>.css` to `presets/<new-name>.css` and edit the token values.
2. Update the table at the top of this SKILL.md with name + feel + accent + notes.
3. Validate by applying the preset to the React demo and rebuilding.
4. The preset CSS file MUST begin with the marker comment so step 5 above's overwrite-detection works:
   ```css
   /* markbook style preset: <name> */
   ```

## Prevention tests

- Preset CSS file ends in `.css`, lives at `presets/<name>.css`, and starts with the marker comment.
- All overrides target `--mb-*` tokens (no rule selectors beyond `:root` / `:root[data-theme="dark"]`). Anything beyond that belongs in Layer 2 (`disableBaseCss`) or Layer 3 (`transformHtml`).
- After applying, `markbook build` succeeds and the rendered HTML contains a `<style data-markbook-user-css>` block referencing one of the preset's tokens (e.g. the accent color hex). Verify with `grep`.
