Package mode

The same story consumed via npm-style imports. The host provides React; the package brings the story + adapter mount + decorator stack.

Setup

cd examples/react-demo
markbook bundle components-button-variants --mode package

Generates ../react-demo/dist/packages/components-button-variants/ — a publishable npm package directory:

components-button-variants/
├── package.json    (peer deps: react, react-dom)
├── README.md
└── dist/
    └── index.js   (~3.5 KB, React is external)

Markup

<script type="importmap">
{
  "imports": {
    "react": "https://esm.sh/react@18.3.1",
    "react-dom": "https://esm.sh/react-dom@18.3.1",
    "components-button-variants": "…/dist/index.js"
  }
}
</script>
<div id="here"></div>
<script type="module">
  import { mount } from 'components-button-variants';
  mount(document.getElementById('here'));
</script>

In a real consumer project, replace the importmap with npm install components-button-variants + a bundler (Vite/Webpack/etc.) that resolves the bare specifiers normally.
