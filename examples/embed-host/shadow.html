Shadow-DOM isolation

The same package-mode bundle, but mounted into an open shadow root via mount(el, { isolation: 'shadow' }). Host-page CSS (an aggressive button reset, see the source of this page) does not reach into the shadow tree — the buttons render with Pixie's intended styles.

Markup

<div id="here"></div>
<script type="module">
  import { mount } from 'components-button-variants';
  mount(document.getElementById('here'), { isolation: 'shadow' });
</script>

How styles get in

The embed bundler bakes the bundle's CSS into a string constant in the entry. At mount time the adapter detects whether the mount target lives inside a shadow root and injects a <style> there instead of in document.head. Token sheets that previously used :root selectors only (like Pixie's pixie.css) gained a paired :host selector so CSS custom properties are visible inside the shadow tree too.

Live

If host-page CSS leaked in, every button would be pink at 30% opacity. They aren't.
